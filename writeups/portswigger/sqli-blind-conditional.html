Blind SQLi — Condicional (SUBSTRING)

Plataforma: PortSwigger Labs Tags: SQLiBlindSUBSTRING

Uso de SUBSTRING con respuestas condicionales

usamos esta función cuando queremos ir obteniendo los valores de la string que retorna una consulta en SQL, uno por uno. Hacemos uso de respuesta condicionales ya que igualamos el resultado de la consulta a algo para saber si es igual o no a eso.

La metodología es simple:

> 1-Buscamos una query de la cual queremos saber el valor que retorna y la concatenamos a la query del servidor con and

> 2-Le pasamos la query a la función LENGTH para saber la longitud de la cadena de respuesta

>3 - Le pasamos esa query como string a la función SUBSTRING()

>3- vamos iterando por posiciones y valores hasta obtener el valor completo del resultado de la consulta

Verificación de vulnerabilidad

accedemos al lab y vemos que hay un trackingID y ademas la pagina nos dice Welcome back eso es porque la Query del lado del servidor si encuentra nuestro TrackingID en la base de datos

Query: SELECT * from users where TRACKINID='gJVOQgCpXxvbKVv8' and substring((SELECT password from users WHERE username='administrator')1,1)='a''

[imagen]

si rompemos la query con un ' al final del TrackID vemos que ya no dice welcome back, por lo tanto, es vulnerable

Explotación

Aquí deberíamos jugar con SUBSTRING tal que asi:

gJVOQgCpXxvbKVv8' and substring((SELECT password from users WHERE username='administrator')1,1)='a'-- -

ya que nos dijeron las columnas y tambien al usuario que debemos buscar la password, lo que hacemos es construir una query que como resultado devuelva la password:

SELECT password from users WHERE username='administrator'

luego hacemos uso de la función SUBSTRING(string,position,len) y le pasamos la Query como string ya que la query devolverá la password y esa será la string.

Luego igualamos todo eso a algo para saber si es verdadero o falso dependiendo el valor y cuando encontremos el valor para la posición uno, podemos ir verificando el de la posición dos.
